# Event Cap

## Goal

Build a mobile app that lets a user capture an image, enter basic item information, and upload everything to the backend for later review.

## Current milestone

Build the first Expo mobile app and connect it to the existing backend upload flow.

## Tech choices

- Backend: Node.js + Express + TypeScript
- Mobile: Expo + React Native + TypeScript
- Storage: backend upload folder already working
- Database: defer until mobile capture flow is proven

## Current scope

The mobile app should:

- capture or select an image
- collect:
  - unique id (auto-generated by the app for now)
  - type
  - description
  - tags
- submit the image and metadata to the existing backend
- show a simple success or failure result

## Constraints

- Keep changes small and easy to review
- Prefer simple libraries and standard Expo approaches
- Do not introduce authentication or cloud hosting yet
- Do not redesign the backend unless required for the upload contract
- Use AI as an assistant, not an uncontrolled coder

## Definition of done

- Expo app created and runs locally
- User can capture or choose an image
- User can enter item fields
- User can submit to backend successfully
- Basic validation exists
- README / setup notes updated
